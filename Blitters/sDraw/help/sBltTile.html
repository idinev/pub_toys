sBltTile

sBltTile performs multiple sBlt() calls, to fill in the TileX:TileY-TileWidth:TileHeight rectangle. The small rectangle to be used as a tile is specified by SpriteX:SpriteY-SpriteWid:SpriteHei. An additional offset is specified by StartX:StartY.


void sBltTile(long TileX,long TileY,long TileWidth,long TileHeight, long SpriteX,long SpriteY,long SpriteWid,long SpriteHei, long StartX,long StartY);

Parameters

TileX,TileY,TileWidth,TileHeight
target rectangle
SpriteX,SpriteY,SpriteWid,SpriteHei
source rectangle
StartX,StartY
a positive or zero offset from the source rectangle. Usually, StartX=0 and StartY=0. Ideally, StartX=0..SpriteWid, StartY=0..SpriteHei

Return Value

None. [ASM: All registers are preserved.]

Example:

sBltTile(0,0,400,300, 0,0,64,64, 0,0);

Remarks

sBltTile is not optimized. It calls sBlt() many times, without getting any hint whether it could stop earlier (due to clipping). Thus, using a tiny source-rectangle generally isn't a good idea. 40x40 source-rectangles and above are best, as regards speed.
