sBlt

sBlt copies a rectangle from an sSprite onto the screen backbuffer (or onto the target sSprite in advanced cases).
Depending on the flags of the source-sSprite, sBlt performs one of 3 types of blending:
1) direct copy - this is by default. The result is visually identical to a GDI's BitBlt(....,SRCCOPY) .
2) alphablend - uses the alpha-channel of the sSprite, if the latter is marked as having such.
3) premultiplied alphablend - same as alphablend, but faster - if the sSprite's alpha-channel is premultiplied.


void sBlt(long x,long y,long wid,long hei,long x2,long y2);

Parameters

x,y,wid,hei,x2,y2
The usual parameters for any Blt-like operation:
x:y specify the top-left corner of the target rectangle
wid:hei specify the width and height of the target rectangle
x2:y2 specify the top-left corner of the source rectangle

Return Value

None. [ASM: All registers are preserved.]

Example:

sBlt(0,0,400,300,0,0);

Remarks

Note that all sBltXXX procedures do not check the parameters x2 and y2 - wrong parameters can lead to a GPF! This means, if your source sSprite is 200x200, the above example code will crash or misbehave.
