sdLoadFont

sdLoadFont loads an sDraw Font from a specified memory location. The loading consists of simply decompressing the data to an internally-allocated array of [header + 8bpp data]


void* sdLoadFont(void* pSDFontData);
extern void* SD_CurFont;

Parameters

pSDFontData
memory location, from which to load the SD Font

Return Value

pointer to the SDFont object. Use this value to set the SD_CurFont variable before drawing text. (SD_CurFont is inside sDraw.lib).

Example:

extern char softImage1[];
void* MyFont;
MyFont = sdLoadFont(&softImage1[0]);
...
SD_CurFont = MyFont;
sTextOut(10,10,"hello world!",0xFFFFFF);

Remarks

In an sDraw font, only 96 ASCII characters are saved- 32..127. They're in raster format, in an 8bpp array, which is actually their alpha-channel (fonts by themselves don't have RGB color). Use the provided tool "Tools\FontMaker.exe" to make SDFonts.
SD fonts have a field in their header, that specifies if the font is 'smooth' (ala WildWords) or 'solid' (ala Fixedsys). This is to gain speed when drawing 'solid' fonts. (used internally by sTextOut)
