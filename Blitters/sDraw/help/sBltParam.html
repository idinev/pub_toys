sBlt***Param procedures

sBlt***Param switch the source sSprite to the sSprite, given in the additional "pSprite" parameter, call the sBlt*** function, and restore to the previous source-sSprite. Switching between textures is a bit slow (16-50 cycles per switch), and relatively rare, thus in sDraw it's brought to minimum.
But in cases when we need to Blt from some other sSprite, and revert to the previous source-sSprite, sBlt***Param are mostly convenient.


List of sBlt***Param procedures:
void sBltParam(long x,long y,long wid,long hei,long x2,long y2,sSprite* pSprite);
void sBltTransParam(long x,long y,long wid,long hei,long x2,long y2,sSprite* pSprite);
void sBltTransAlphaParam(long x,long y,long wid,long hei,long x2,long y2,long Alpha,sSprite* pSprite);
void sBltAlphaParam(long x,long y,long wid,long hei,long x2,long y2,long Alpha,sSprite* pSprite);
void sBltAddParam(long x,long y,long wid,long hei,long x2,long y2,sSprite* pSprite);
void sBltAddFadeParam(long x,long y,long wid,long hei,long x2,long y2,long Fader,sSprite* pSprite);
void sBltTintParam(long x,long y,long wid,long hei,long x2,long y2,long dwColor,sSprite* pSprite);

Return Value

None. [ASM: All registers are preserved.]

Example:

sBltParam(0,0,400,300,0,0,pBackgroundSprite);

sBltAddFadeParam(0,0,400,300,0,0, 166, pLightSprite);

Remarks

Total overhead per call is 40-120 cycles. When several consequent sBlt*** calls, that use the same sSprite, are present in your code, you should use sdSetSourceSprite.
